# 高校班级事务管理系统 - 项目总结

## 项目概述

高校班级事务管理系统是一个基于C++17开发的Web应用程序，提供完整的班级管理功能，包括用户管理、通知管理、班级信息管理、学习管理、班费管理、纪律管理和活动管理等模块。

## 已完成的功能

### 1. 项目架构设计 ✅
- **分层架构**: 采用清晰的分层设计，包括Web服务层、业务逻辑层、数据访问层和数据库层
- **模块化设计**: 每个功能模块独立开发，便于维护和扩展
- **松耦合**: 模块间通过接口通信，降低耦合度

### 2. 数据库设计 ✅
- **完整的数据库Schema**: 包含14个核心数据表
- **规范化设计**: 遵循第三范式，确保数据完整性
- **索引优化**: 为常用查询字段创建索引
- **外键约束**: 保证数据关系完整性
- **视图和存储过程**: 提供高级数据操作功能

### 3. 核心模块实现 ✅

#### 3.1 用户管理模块
- 用户注册、登录、登出功能
- 密码加密存储（SHA256 + Salt）

#### 3.2 通知管理模块

- 阅读状态跟踪
- 未读通知统计

- 批量通知操作

#### 3.3 数据库访问层
- 数据库连接池管理
- 事务处理
- 预处理语句支持
- 错误处理和日志记录

#### 3.4 Web服务层
- 基于Crow框架的RESTful API
- 路由管理
- 请求/响应处理

### 4. 工具类库 ✅

- **配置管理**: JSON配置文件加载和管理

### 5. 项目管理 ✅

## 核心API接口

### 用户管理API
- `POST /api/v1/users/login` - 用户登录
- `POST /api/v1/users/register` - 用户注册
- `GET /api/v1/users/{id}` - 获取用户信息
- `PUT /api/v1/users/{id}` - 更新用户信息
- `PUT /api/v1/users/{id}/password` - 修改密码
- `GET /api/v1/users` - 获取用户列表

### 通知管理API
- `POST /api/v1/notifications` - 发布通知
- `GET /api/v1/notifications` - 获取通知列表
- `GET /api/v1/notifications/{id}` - 获取通知详情
- `PUT /api/v1/notifications/{id}` - 更新通知
- `DELETE /api/v1/notifications/{id}` - 删除通知
- `POST /api/v1/notifications/{id}/read` - 标记为已读
- `GET /api/v1/notifications/unread` - 获取未读通知
- `GET /api/v1/notifications/unread/count` - 获取未读数量

## 构建和部署

### 构建要求
- C++17编译器 (GCC 7.0+ 或 Clang 5.0+)
- CMake 3.10+
- MySQL 5.7+
- OpenSSL
- Crow C++ Web框架
- Google Test (用于测试)

### 构建步骤
```bash
# 1. 创建构建目录
mkdir build && cd build

# 2. 配置CMake
cmake .. -DCMAKE_BUILD_TYPE=Release

# 3. 编译
make -j4

# 4. 运行测试
make test

# 5. 安装
sudo make install
```

### 运行步骤
```bash
# 1. 设置数据库
./scripts/setup_database.sh

# 2. 构建项目
./scripts/build.sh --type Release

# 3. 运行系统
./scripts/run.sh --host 0.0.0.0 --port 8080
```
