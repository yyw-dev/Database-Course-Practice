# 高校班级事务管理系统需求分析

## 1. 项目背景

随着高校班级管理事务逐渐增多，传统微信群、QQ 群等沟通方式在信息传达、事务处理、数据统计和历史归档方面存在不足。例如通知容易被刷屏、文件资料难以长期检索、班费收支缺乏统一记录、评优评奖过程不够透明、请假和活动报名数据分散等。

为提高班级事务管理效率，促进班级管理信息化建设，本项目拟设计并实现“高校班级事务管理系统”。系统面向高校自然班级，以班级成员、通知公告、学习事务、班费收支、纪律记录、活动报名、评优投票、请销假等数据为核心，提供统一的数据存储、权限控制、查询统计和事务管理能力。

## 2. 项目目标

本系统旨在为大学班级提供一体化、数字化的管理平台，主要目标如下：

1. 规范化管理：将班级事务流程化、标准化，减少重复性人工统计与口头通知。
2. 信息透明化：实现班费收支、重要通知、活动安排、评优结果等信息的公开透明。
3. 协同高效化：为班主任、班长、职能班委和普通同学提供清晰的协作入口。
4. 数据资产化：集中保存班级成员、学习、财务、纪律、活动等数据，形成可追溯的班级数字档案。
5. 查询统计便捷化：支持按班级、成员、时间、事务类型等维度进行检索和统计分析。

## 3. 目标用户

系统主要面向以下用户：

| 用户类型 | 用户说明 | 典型使用场景 |
| --- | --- | --- |
| 班主任/辅导员 | 班级事务的监督者和决策者 | 审核关键事项、查看统计报表、发布重要通知 |
| 班长/团支书等核心班委 | 班级事务的主要管理者 | 管理班级信息、成员账号、通知活动、角色权限 |
| 学习委员 | 学习管理模块负责人 | 维护课程、课表、成绩、作业或考试信息 |
| 生活/财务委员 | 班费管理模块负责人 | 记录班费收入支出、公示明细、发起大额支出审核 |
| 纪律/纪检委员 | 纪律管理模块负责人 | 登记考勤、违纪、奖惩等记录 |
| 班级普通成员 | 信息查看者和事务参与者 | 查看通知、班费、成绩统计，参与投票活动，提交请假申请 |

## 4. 业务范围

系统围绕高校班级日常事务管理，主要业务范围包括：

1. 班级与成员管理：维护班级基本信息、学生信息、班委角色、账号状态。
2. 通知公告管理：发布、查询、确认阅读班级通知。
3. 学习事务管理：维护课程、课表、成绩、学习资料和学习统计数据。
4. 班费管理：记录班费收入、支出、余额、经办人、审核状态和附件凭证。
5. 活动管理：发布活动、管理报名、记录活动结果。
6. 请销假管理：学生提交请假申请，班委或班主任审核，记录销假状态。
7. 纪律管理：记录考勤、违纪、奖惩等纪律数据，支持统计查询。
8. 评优投票管理：组织评优评奖、民主投票、候选人管理和结果公示。
9. 统计报表：按模块生成班级事务数据统计结果。
10. 权限与日志：按角色控制数据访问范围，并记录关键操作日志。

## 5. 用户角色与权限需求

### 5.1 角色定义

| 角色 | 权限描述 |
| --- | --- |
| 班主任 | 拥有系统最高权限；审核大额班费支出、最终评优名单、重大纪律处分等关键操作；查看所有模块统计数据和报表；发布面向全班的重要通知 |
| 班长 | 权限仅次于班主任；管理班级基本信息、成员账号与角色分配；统筹发布通知和活动；监督各模块运行；拥有大部分功能的新增、删除、修改、查询权限 |
| 团支书 | 协助班长管理团务、活动、评优评奖等事务；可参与通知、活动和投票管理 |
| 学习委员 | 负责学习管理模块的数据录入与维护，包括课程、课表、成绩、学习资料等 |
| 生活/财务委员 | 负责班费管理模块，包括班费收支记账、公示和凭证维护 |
| 纪律/纪检委员 | 负责纪律管理模块，包括考勤、违纪、奖惩等记录与统计 |
| 班级成员 | 查看个人和班级公开信息；查看班费明细、通知、课表、成绩统计；参与投票、活动报名；提交请假和销假申请 |

### 5.2 权限控制原则

1. 普通成员只能查看公开信息和本人的私有数据。
2. 职能班委只能维护其职责范围内的数据。
3. 班长可管理班级基础信息和大部分班级事务数据。
4. 班主任拥有最终审核权和全局查看权限。
5. 涉及财务、评优、处分等关键数据的修改应保留操作日志。
6. 删除重要业务数据时宜采用逻辑删除或状态变更，避免历史记录丢失。

## 6. 功能需求

### 6.1 登录与用户管理

#### 6.1.1 用户登录
1. 用户在登录页面输入学号/账号和密码。
2. 系统验证输入信息合法性（非空、格式正确）。
3. 系统查询用户表，匹配学号/账号和密码摘要。
4. 若匹配成功，检查账号状态（是否停用）。
5. 若账号正常，生成会话令牌，记录登录时间和日志。
6. 返回用户信息、所属班级、当前角色列表。
7. 若登录失败，返回错误提示（账号不存在/密码错误/账号已停用）。

#### 6.1.2 用户信息维护
1. 班长或班主任进入用户管理页面，查询班级成员列表。
   - 支持按学号、姓名、角色、加入时间筛选。
   - 支持分页展示。
2. 新增成员：
   - 录入学号、姓名、性别、手机号、邮箱等基础信息。
   - 系统验证学号唯一性。
   - 生成默认密码或发送激活链接。
   - 创建用户角色记录（初始为班级成员）。
   - 记录操作日志。
3. 修改成员信息：
   - 选择成员，编辑允许修改的字段。
   - 系统更新用户表和班级成员表。
   - 记录修改前后的关键字段变更日志。
4. 停用成员账号：
   - 班长或班主任选择成员，点击停用。
   - 系统验证操作权限。
   - 更新账号状态为"已停用"。
   - 记录停用时间和操作者。
   - 返回停用成功提示。
5. 分配角色：
   - 选择成员和要赋予的角色（可多选）。
   - 系统检查角色合法性和权限冲突。
   - 插入用户角色关联记录，记录授权时间。
   - 记录角色变更日志。

#### 6.1.3 成员查询
1. 用户进入成员列表页面。
2. 输入或选择查询条件（学号、姓名、角色、加入状态）。
3. 系统执行多条件查询，返回符合条件的成员列表。
4. 支持导出成员信息（仅班长及以上权限）。

### 6.2 班级信息管理

#### 6.2.1 班级信息查看
1. 普通成员访问班级信息页面。
2. 系统读取当前用户所属班级信息。
3. 返回班级名称、学院、专业、年级、班主任、班委列表等公开信息。

#### 6.2.2 班级信息修改
1. 班长或班主任进入班级管理页面。
2. 展示当前班级的完整信息。
3. 编辑班级名称、学院、专业、班主任、班级简介等字段。
4. 系统验证输入数据有效性。
5. 保存修改，更新班级表。
6. 记录修改操作日志（记录修改字段、修改前后值）。
7. 返回修改成功提示。

#### 6.2.3 班委信息管理
1. 班长或班主任进入班委管理页面。
2. 系统查询本班所有具有班委角色的成员。
3. 支持查看各班委的职务、联系方式、权限范围。
4. 支持更新班委信息、变更班委职务或撤销。
5. 新增班委：选择成员和班委职务，系统自动分配相应角色。
6. 记录班委变更日志。

### 6.3 通知公告管理

#### 6.3.1 通知发布
1. 班主任、班长或授权班委点击发布通知。
2. 在通知表单中输入：
   - 标题、内容（支持富文本编辑）
   - 通知类型（学习、生活、纪律、财务、其他）
   - 重要程度（普通、重要、紧急）
   - 有效状态（正常、已过期、已删除）
   - 上传附件（可选）
3. 系统验证必填字段。
4. 创建通知记录，自动填充发布人、发布时间。
5. 系统向本班所有成员创建通知订阅记录（初始为未读）。
6. 记录发布日志。
7. 返回发布成功，可查看通知详情。

#### 6.3.2 通知查看与已读记录
1. 成员进入通知列表页面。
2. 系统查询该成员所在班级的所有有效通知。
3. 按发布时间倒序排列，支持按类型、重要程度筛选。
4. 成员点击通知查看详情。
5. 系统检查是否已生成该用户的阅读记录。
6. 若无阅读记录，创建新的阅读记录，记录阅读时间。
7. 若已有记录，更新阅读状态为已读。
8. 返回通知详情和阅读时间。

#### 6.3.3 通知已读统计
1. 通知发布者进入通知详情页面。
2. 点击已读统计按钮。
3. 系统查询该通知的所有阅读记录。
4. 统计已读人数、未读人数、已读率。
5. 支持导出未读成员名单。
6. 返回统计结果和成员列表。

#### 6.3.4 通知查询
1. 用户在通知列表进行搜索：
   - 按发布时间范围查询。
   - 按通知类型过滤。
   - 按重要程度过滤。
   - 按关键字搜索标题和内容。
   - 按发布人筛选。
2. 系统执行多条件查询，返回符合条件的通知列表。
3. 支持排序（时间、类型、重要程度）。

### 6.4 学习管理

#### 6.4.1 课程维护
1. 学习委员进入课程管理页面。
2. 新增课程：
   - 输入课程名称、任课教师、课程类型、学分等。
   - 系统验证课程名称不重复。
   - 创建课程记录，自动分配课程编号。
   - 记录新增日志。
3. 修改课程信息：
   - 选择课程，编辑课程详情。
   - 系统更新课程表。
   - 记录修改日志。
4. 删除课程：
   - 系统检查该课程是否关联课表或成绩记录。
   - 若无关联，允许删除；否则提示不可删除。
   - 记录删除日志。

#### 6.4.2 课表管理
1. 学习委员进入课表管理页面。
2. 新增课表记录：
   - 选择课程，输入班级、上课时间、上课地点。
   - 系统验证时间和地点信息有效性。
   - 创建课表记录。
   - 记录新增日志。
3. 修改课表：
   - 编辑上课时间、地点等信息。
   - 系统更新课表表。
   - 记录修改日志。

#### 6.4.3 课表查询
1. 成员进入课表查询页面。
2. 系统根据学生班级自动获取本班课表。
3. 支持按课程名称、上课地点、时间段筛选。
4. 返回课程列表，显示课程名称、任课教师、上课时间、地点。
5. 支持导出课表（PDF、Excel格式）。

#### 6.4.4 成绩录入
1. 学习委员进入成绩管理页面。
2. 选择课程和考试类型（期中、期末、平时等）。
3. 批量录入成绩：
   - 系统生成该课程的班级学生列表。
   - 学习委员逐行输入或粘贴学号和成绩。
   - 系统验证学号存在性和成绩有效性（0-100或等级）。
4. 系统创建成绩记录，自动填充录入人、录入时间。
5. 支持导入成绩（Excel、CSV格式）：
   - 上传文件后，系统解析并验证格式和数据。
   - 显示解析结果和错误提示。
   - 确认无误后批量插入成绩表。
6. 记录成绩录入日志。

#### 6.4.5 成绩查看
1. 普通成员进入成绩查询页面。
2. 系统返回该学生的个人成绩列表（课程、成绩、考试类型、录入时间）。
3. 班长、班主任、学习委员进入成绩统计页面。
4. 系统执行聚合查询，返回班级成绩统计：
   - 课程维度统计：平均分、最高分、最低分、及格率、不及格人数。
   - 学生维度排名：学生成绩列表，支持按平均分排序。
   - 成绩分布统计：按分数段（优秀、良好、及格、不及格）统计人数。
5. 支持按课程、考试类型、成绩段等条件筛选。
6. 支持导出统计报表。

### 6.5 班费管理

#### 6.5.1 班费收入/支出记录
1. 生活/财务委员进入班费管理页面。
2. 新增班费记录：
   - 选择收支类型（收入/支出）。
   - 输入金额、事由、发生时间、附件（凭证图片）。
   - 若为支出，输入支出用途详情。
   - 系统自动填充经办人（当前登录用户）和记录时间。
3. 系统验证金额大于0、时间有效。
4. 创建班费流水记录，初始审核状态：
   - 收入记录：状态为"已确认"。
   - 普通支出（金额）：状态为"已确认"。
   - 大额支出（≥500元或自定义阈值）：状态为"待审核"。
5. 记录班费操作日志。
6. 返回记录成功提示。

#### 6.5.2 班费余额计算
1. 系统设置班费初始余额（如1000元）。
2. 每次班费记录插入时，触发自动计算：
   - 查询所有状态为"已确认"的班费流水。
   - 按发生时间累加收入，减去支出。
   - 更新班级表的余额字段。
3. 每日或定期更新一次余额统计。

#### 6.5.3 班费支出审核
1. 班主任进入班费审核页面。
2. 系统查询所有状态为"待审核"的班费记录。
3. 班主任选择待审核记录，查看详情：
   - 支出金额、事由、凭证、经办人等。
4. 班主任进行审核操作：
   - 点击"同意"或"驳回"。
   - 输入审核意见（可选）。
5. 系统更新记录状态：
   - 审核状态为"已通过"或"已驳回"。
   - 记录审核人、审核时间、审核意见。
6. 若审核通过，班费余额在下次计算时生效。
7. 记录审核日志。

#### 6.5.4 班费查看与公示
1. 普通成员进入班费页面。
2. 系统查询本班所有状态为"已确认"的班费流水。
3. 返回班费流水列表，显示：
   - 流水编号、发生时间、收支类型、金额、事由、经办人。
4. 支持按时间、收支类型、经办人筛选。
5. 页面顶部显示当前班费余额。
6. 支持导出班费明细表（PDF、Excel格式）。

#### 6.5.5 班费统计查询
1. 班长、班主任进入班费统计页面。
2. 系统提供多维度统计：
   - 时间段统计：某月/某季度/某年的收入、支出、余额变化。
   - 支出类型统计：各类支出金额和占比（饼图）。
   - 经办人统计：各人经办的收支金额汇总。
   - 审核状态统计：已确认、待审核、已驳回的记录数和金额。
3. 支持条件筛选：时间范围、收支类型、审核状态。
4. 支持导出统计报表。

### 6.6 活动管理

#### 6.6.1 活动发布
1. 班长、团支书或授权班委进入活动发布页面。
2. 输入活动信息：
   - 活动名称、活动类型、活动说明。
   - 活动时间（开始、结束）、活动地点。
   - 报名截止时间、人数限制。
   - 活动海报或相关附件。
3. 系统验证时间逻辑（报名截止 < 活动开始 < 活动结束）。
4. 创建活动记录，初始状态为"进行中"或"报名中"。
5. 自动填充发布人、发布时间。
6. 向本班所有成员发送活动通知。
7. 记录活动发布日志。

#### 6.6.2 活动报名
1. 普通成员进入活动列表页面。
2. 系统显示所有进行中的活动名称
3. 成员选择活动，查看详情（名称、时间、地点、报名人数等）。
4. 点击报名，系统检查：
   - 是否在报名截止时间内。
   - 活动是否未满人数。
   - 成员是否已报名过。
5. 检查通过后，创建活动报名记录，记录报名时间、报名状态为"已报名"。
6. 更新活动的报名人数。
7. 返回报名成功提示。

#### 6.6.3 取消报名
1. 已报名的成员点击取消报名。
2. 系统检查报名是否在截止时间内（逻辑上应允许取消）。
3. 删除或更新报名记录的状态为"已取消"。
4. 更新活动报名人数。
5. 返回取消成功提示。

#### 6.6.4 活动结果记录
1. 活动结束后，发布者进入活动详情页面。
2. 点击"活动总结"，输入：
   - 实际参与人数。
   - 活动总结、成果描述。
   - 活动照片或相关附件。
3. 系统创建或更新活动结果记录。
4. 更新活动状态为"已结束"。
5. 记录活动总结日志。

#### 6.6.5 活动报名统计
1. 活动发布者进入活动管理页面。
2. 选择活动，点击报名统计。
3. 系统返回：
   - 报名总人数、确认参与人数、未参与人数。
   - 报名成员名单（学号、姓名、报名时间）。
4. 支持导出报名名单。

### 6.7 请销假管理

#### 6.7.1 请假申请提交
1. 学生进入请假申请页面。
2. 输入请假信息：
   - 请假类型（事假、病假、公假、其他）。
   - 请假开始时间、结束时间。
   - 请假原因、证明材料（上传证明文件）。
3. 系统验证时间逻辑（结束时间 > 开始时间）。
4. 创建请假申请记录，初始状态为"待审核"。
5. 自动填充申请人和申请时间。
6. 向班委或班主任发送待审核通知。
7. 记录请假申请日志。

#### 6.7.2 请假申请审核
1. 班委或班主任进入请假审核页面。
2. 系统查询所有状态为"待审核"的请假申请。
3. 选择申请，查看详情（申请人、请假类型、时间、原因、证明材料）。
4. 进行审核：
   - 点击"通过"或"驳回"。
   - 输入审核意见（可选）。
5. 系统更新申请状态：
   - 状态为"已通过"或"已驳回"。
   - 记录审核人、审核时间、审核意见。
6. 向申请人发送审核结果通知。
7. 记录审核日志。

#### 6.7.3 销假记录
1. 请假已通过的学生，在假期结束后进入销假页面。
2. 系统显示该学生的所有"已通过"且尚未销假的请假申请。
3. 学生选择申请，点击销假。
4. 系统检查销假时间是否晚于请假结束时间。
5. 创建销假记录，记录销假时间、销假状态为"已销假"。
6. 更新关联的请假申请状态为"已销假"。
7. 记录销假日志。

#### 6.7.4 请假数据查询
1. 班长、班主任进入入请假统计页面。
2. 支持多维度查询：
   - 按学生查询：某学生的请假历史（次数、总时长、类型）。
   - 按时间范围查询：某时期内的请假申请。
   - 按请假类型统计：各类请假的人数和时长。
   - 按审核状态筛选：待审核、已通过、已驳回、已销假。
3. 返回请假数据列表或统计报表。
4. 支持导出请假数据。

### 6.8 纪律管理

#### 6.8.1 纪律记录登记
1. 纪律/纪检委员进入纪律管理页面。
2. 新增纪律记录：
   - 选择学生。
   - 选择记录类型（考勤、违纪、奖惩）。
   - 输入发生时间、发生地点、具体说明。
   - 对于违纪/奖惩，输入处理状态（待审核、已处理、已撤销）和处理意见。
3. 系统验证学生存在性和时间有效性。
4. 创建纪律记录，自动填充登记人和登记时间。
5. 若涉及重大处分（如严重违纪），状态初始为"待审核"。
6. 记录纪律登记日志。

#### 6.8.2 纪律记录审核
1. 班主任进入纪律审核页面。
2. 系统查询所有状态为"待审核"的纪律记录。
3. 选择记录，查看详情（学生、记录类型、发生情况、处理建议）。
4. 进行审核：
   - 点击"通过"、"驳回"或"需补充"。
   - 输入审核意见。
5. 系统更新记录状态为"已处理"或"已驳回"。
6. 记录审核人、审核时间、审核结果。
7. 记录审核日志。

#### 6.8.3 纪律记录查看
1. 普通成员进入纪律查询页面。
2. 系统仅返回该学生本人的纪律记录（隐私保护）。
3. 显示考勤情况、违纪记录、奖励记录。
4. 班长、纪律委员、班主任可查看本班所有成员的纪律记录。

#### 6.8.4 纪律统计
1. 班长、班主任、纪律委员进入纪律统计页面。
2. 系统提供多维度统计：
   - 考勤统计：缺勤人数、缺勤率、迟到人次等。
   - 违纪统计：各类违纪人数、违纪次数。
   - 奖励统计：获奖学生名单、奖励类型和数量。
   - 学生排名：按纪律记录数（或缺勤次数）排序。
3. 支持按时间段、记录类型、处理状态筛选。
4. 支持导出统计报表。

### 6.9 评优投票管理

#### 6.9.1 评优项目创建
1. 班长、团支书或班主任进入评优管理页面。
2. 新增评优项目：
   - 输入项目名称、说明、评选规则。
   - 设置投票开始时间、结束时间、候选人范围。
   - 定义候选人数量和名额限制。
   - 选择投票规则（单选/多选、单次投票/多次投票）。
3. 系统验证时间逻辑。
4. 创建评优项目记录，初始状态为"进行中"或"报名中"。
5. 自动填充创建人和创建时间。
6. 记录项目创建日志。

#### 6.9.2 候选人管理
1. 有权管理评优项目的用户进入项目详情页面。
2. 支持两种方式添加候选人：
   - 候选人报名：系统开放报名入口，符合条件的学生可自行报名。
     1. 学生进入投票页面，选择项目，点击报名。
     2. 输入报名理由、推荐材料。
     3. 系统创建候选人记录，初始审核状态为"待审核"。
   - 管理员提名：班长或班主任直接添加候选人。
     1. 选择学生，输入推荐理由。
     2. 系统创建候选人记录，审核状态为"待审核"。
3. 班主任审核候选人：
   - 查看所有待审核的候选人报名。
   - 点击"通过"或"驳回"。
   - 系统更新候选人审核状态。

#### 6.9.3 投票过程
1. 投票时间开始后，班级成员可进入投票页面。
2. 系统验证成员是否已投票（根据投票规则）。
3. 显示该项目的所有审核通过的候选人（候选人名单、推荐理由）。
4. 根据投票规则（单选/多选）：
   - 单选：成员选择一名候选人，点击投票。
   - 多选：成员可选择多名候选人，确认投票。
5. 系统验证投票有效性，防止重复投票（若规则不允许）。
6. 创建投票记录（项目编号、投票人、被投票候选人、投票时间）。
7. 返回投票成功提示。

#### 6.9.4 投票结果统计
1. 投票时间结束后，班长或班主任进入投票统计页面。
2. 系统自动统计结果：
   - 候选人得票数、得票率。
   - 投票参与率（投票人数 / 班级总人数）。
   - 候选人排名（按得票数从高到低）。
3. 显示最终排名名单（含并列情况）。
4. 支持导出投票统计报表。

#### 6.9.5 评优结果审核与公示
1. 班主任进入评优审核页面。
2. 查看投票统计结果。
3. 进行最终审核：
   - 确认评选结果或进行调整（如对并列候选人的决策）。
   - 输入审核意见。
4. 系统创建评优结果记录，状态为"已公示"。
5. 记录审核人、审核时间。
6. 向班级成员发送评优结果公示通知。
7. 普通成员可查看最终的评优结果（公示名单、推荐理由）。
8. 记录评优结果日志。

### 6.10 统计报表

#### 6.10.1 班费统计报表
1. 班长、班主任进入班费报表页面。
2. 生成报表：
   - 按月/季/年统计班费收入、支出、余额变化。
   - 按支出类型统计各类支出金额和占比。
   - 显示支出趋势图表。
3. 支持时间范围筛选和导出。

#### 6.10.2 通知已读率报表
1. 班长进入通知统计页面。
2. 系统查询本班所有通知。
3. 对每条通知统计：
   - 已读人数、未读人数、已读率。
4. 返回通知已读率列表，支持排序和筛选。
5. 支持导出报表。

#### 6.10.3 活动报名统计报表
1. 班长、班主任进入活动统计页面。
2. 系统查询本班所有已结束的活动。
3. 对每个活动统计：
   - 报名人数、实际参与人数、参与率。
   - 性别分布、班级分布等维度统计。
4. 支持按时间、活动类型筛选。
5. 支持导出报表。

#### 6.10.4 请假统计报表
1. 班长、班主任进入请假统计页面。
2. 系统提供多维度统计：
   - 班级请假总次数、总时长、请假率。
   - 按请假类型统计次数和时长。
   - 学生排名：按请假次数或时长排序。
3. 支持时间范围和请假类型筛选。
4. 支持导出报表。

#### 6.10.5 纪律统计报表
1. 班长、班主任、纪律委员进入纪律报表页面。
2. 系统提供统计：
   - 班级总缺勤人次、缺勤率。
   - 违纪人数、各类违纪次数。
   - 获奖学生名单和奖励类型。
   - 学生纪律排名（按违纪次数或缺勤次数）。
3. 支持时间范围和纪律类型筛选。
4. 支持导出报表。

#### 6.10.6 成绩统计报表
1. 班长、班主任、学习委员进入成绩报表页面。
2. 系统提供多维度统计：
   - 课程维度：各课程平均分、最高分、最低分、及格率。
   - 成绩分布：按优秀、良好、及格、不及格统计人数和占比。
   - 学生排名：按学科平均分排序。
3. 支持按课程、考试类型、成绩段筛选。
4. 支持导出统计报表。

### 6.11 操作日志

#### 6.11.1 日志记录
1. 系统在以下关键操作时自动记录日志：
   - 用户管理：新增、修改、停用成员。
   - 班级信息修改。
   - 通知发布、删除。
   - 班费记录新增、审核。
   - 活动发布、活动结果记录。
   - 请假申请、审核、销假。
   - 纪律记录登记、审核。
   - 评优项目创建、结果公示。
   - 投票记录。
   - 成绩录入、修改。
   - 角色分配变更。
2. 每条日志包含：
   - 日志编号、操作人、操作时间。
   - 操作对象（如操作的通知编号、班费流水编号等）。
   - 操作类型（新增、修改、删除、审核、发布、公示等）。
   - 操作详情（修改前后值、审核意见等）。
   - 操作结果（成功、失败）。
   - 操作者IP、设备信息（可选）。

#### 6.11.2 日志查询与审计
1. 班主任或系统管理员进入操作日志页面。
2. 支持多条件查询：
   - 按操作人、操作类型、操作对象、操作时间筛选。
   - 按操作结果（成功/失败）过滤。
3. 返回日志列表，支持查看日志详情。
4. 支持按操作类型、操作人进行分析统计。
5. 支持导出日志记录。

| 投票记录 | 成员投票信息 | 项目编号、投票人、候选人、投票时间 |
| 操作日志 | 系统关键操作记录 | 日志编号、操作人、操作对象、操作类型、操作时间 |

### 7.2 主要数据关系

1. 一个班级可以包含多个班级成员，一个用户通常属于一个主要班级。
2. 一个用户可以拥有多个角色，一个角色可以分配给多个用户。
3. 一个班级可以发布多条通知，一条通知可以被多个成员阅读。
4. 一个班级可以开设多门课程，一门课程可以对应多条课表记录。
5. 一个学生可以拥有多条成绩记录，每条成绩对应一门课程。
6. 一个班级拥有多条班费流水，每条流水由一个用户经办，可由班主任审核。
7. 一个活动可以有多个报名记录，一个成员可以报名多个活动。
8. 一个请假申请由一个学生提交，可对应一条或多条审核、销假记录。
9. 一个评优项目包含多个候选人和多条投票记录。
10. 所有关键业务操作应关联操作日志，便于审计和追踪。

## 8. 数据完整性与约束需求

1. 用户学号在系统中应保持唯一。
2. 班级编号、课程编号、通知编号、活动编号等主键字段应唯一且非空。
3. 外键关联应保证数据一致性，例如成绩记录必须关联有效学生和课程。
4. 金额字段应大于 0，班费流水应区分收入和支出。
5. 请假结束时间应晚于开始时间。
6. 活动报名时间不得晚于报名截止时间。
7. 投票记录应防止同一用户在同一评优项目中重复投票，除非项目规则允许多选。
8. 重要数据删除应采用状态字段标记，不宜直接物理删除。
9. 审核类数据应包含审核人、审核时间和审核结果。

## 9. 非功能需求

### 9.1 安全性

1. 用户密码不得明文存储，应保存密码摘要。
2. 系统应根据角色进行访问控制，防止越权访问。
3. 涉及个人成绩、纪律、请假原因等敏感信息时，应限制查看范围。
4. 财务、评优、处分等关键操作应记录日志。

### 9.2 可用性

1. 常用查询应支持按关键字和条件筛选。
2. 信息列表应支持分页展示。
3. 表单输入应进行合法性校验，并给出明确错误提示。
4. 系统操作流程应符合班级日常事务处理习惯。

### 9.3 可维护性

1. 数据库表结构应命名清晰，字段含义明确。
2. 业务模块之间应保持适当独立，便于后续扩展。
3. 关键字段应建立索引，如学号、班级编号、发布时间、审核状态等。
4. 系统应保留必要的创建时间、更新时间和状态字段。

### 9.4 性能需求

1. 常用列表查询应在合理时间内返回结果。
2. 统计类查询应尽量利用索引或预聚合方式优化。
3. 对通知、成绩、班费流水等高频查询数据应设计合适索引。

## 10. 业务流程需求

### 10.1 通知发布流程

1. 班主任、班长或授权班委创建通知。
2. 系统保存通知内容并发布给指定班级成员。
3. 成员查看通知后生成阅读记录。
4. 发布者查看已读和未读统计。

### 10.2 班费支出审核流程

1. 生活/财务委员录入支出记录。
2. 若金额达到大额支出标准，则提交班主任审核。
3. 班主任审核通过后，记录生效并计入班费余额。
4. 支出记录向班级成员公示。

### 10.3 请假审批流程

1. 学生提交请假申请。
2. 审核人查看申请内容和证明材料。
3. 审核人给出通过或驳回结果。
4. 学生返校后提交销假。
5. 系统保存完整请销假记录。

### 10.4 评优投票流程

1. 班长、团支书或班主任创建评优项目。
2. 系统维护候选人名单。
3. 班级成员在规定时间内投票。
4. 系统统计投票结果。
5. 班主任审核最终名单。
6. 评优结果公示。

## 11. 数据库设计关注点

1. 角色权限建议采用“用户-角色-权限”或简化的“用户-角色”模型，便于扩展班委分工。
2. 班费流水需要保留审核状态，避免未审核支出直接影响公开余额。
3. 通知阅读、活动报名、投票记录属于多对多关系，应单独建表。
4. 成绩、请假、纪律等涉及个人隐私的数据应与公开信息区分。
5. 操作日志应独立存储，避免业务数据变更后丢失审计线索。
6. 各业务表建议保留 `created_at`、`updated_at`、`status` 等字段，便于追踪和软删除。

## 12. 需求优先级

| 优先级 | 模块 | 说明 |
| --- | --- | --- |
| 高 | 用户与角色管理 | 系统权限控制基础 |
| 高 | 班级成员管理 | 所有业务数据的主体基础 |
| 高 | 通知公告管理 | 班级事务信息发布核心 |
| 高 | 班费管理 | 体现信息透明化的重要模块 |
| 高 | 请销假管理 | 班级日常事务高频流程 |
| 中 | 学习管理 | 支持课表、成绩和学习数据统计 |
| 中 | 活动管理 | 支持活动发布、报名和统计 |
| 中 | 纪律管理 | 支持考勤、违纪和奖惩记录 |
| 中 | 评优投票管理 | 支持评优评奖流程透明化 |
| 低 | 高级统计报表 | 可在基础功能完成后扩展 |

## 13. 需求边界

本课程设计项目重点关注数据库建模、数据操作和事务管理能力，因此系统边界定义如下：

1. 系统主要服务单个或多个高校班级的日常事务管理，不覆盖学校教务系统的全部功能。
2. 成绩数据可由班委录入或导入，不要求直接对接学校教务系统。
3. 班费管理只记录班级内部收支流水，不直接对接真实支付平台。
4. 通知公告只负责班级内部发布与阅读统计，不替代即时通讯软件。
5. 评优投票结果仅作为班级内部评议依据，最终结果仍需班主任或辅导员审核确认。
